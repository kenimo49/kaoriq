# KaoriQ 戦略ドキュメント

## コンセプト

「あなたの性格に合う香りを、データで見つける」

persona-manager（性格診断）× 59,000本の香水データベース × Shopifyドロップシッピングで、パーソナライズドフレグランスECを構築する。個人ブランドに依存しない、仕組みで回る自律EC。

## 2軸戦略における位置づけ

```
軸1: 個人ブランド（kenimoto）
     Tech記事 → 書籍 → 動画コース
     売れる理由 = 「kenさんが作ったから」

軸2: 仕組み（kaoriq）← 本プロジェクト
     フレグランスサイト → LLMO + Pinterest → Shopify
     売れる理由 = 「仕組みが回っているから」
     個人の名前は出なくてもいい
```

## なぜフレグランス（香り雑貨）か

3条件の同時充足で逆算選定:

| 条件 | フレグランス |
|------|------------|
| 高DAプラットフォームで記事投稿可能 | Pinterest + Medium |
| LLMが正確に記事を書ける | 香料成分・論文ベースで検証可能 |
| ドロップシッピング利益率 | 40-60% |

追加優位性:
- 法規制: 空間フレグランス（キャンドル、ディフューザー、お香）は雑貨扱いで許認可不要
- LLMO競合: フレグランス領域でLLMO最適化している競合はほぼゼロ
- 既存資産: persona-manager、書籍3冊、harness-ops、generate-image すべて転用可能

## 競合環境の評価

### デザイン面
- Fragrantica、Basenotes、CaFleureBon等の大手は2010年代のデザインのまま
- サイドバー広告、雑然レイアウト、モバイル対応が中途半端
- kaoriqは初日からモダンデザイン + ダーク/ライト + レスポンシブ

### LLMO面
- llms.txt設置サイト: ほぼゼロ
- JSON-LD構造化データ: 少ない
- 参入余地が極めて大きい

### コンテンツ面
- 大手は人力更新で月数本ペース
- kaoriqはharness-opsで1日1-2本 × 4PF = 月120投稿
- 3ヶ月で記事数で追いつく計算

## ビジネスモデル

### ドロップシッピング（海外サプライヤー）

```
顧客がkaoriq.comで購入
  → DSers/CJdropshippingが自動でAliExpress等に注文転送
  → サプライヤーが顧客に直接発送
```

kenさんが仕入れ先と連絡する必要なし。注文転送・支払い・追跡番号取得はすべて自動。

### 配送

| 顧客 | 発送元 | 配送日数 | ブランディング |
|------|--------|---------|--------------|
| 海外（EN圏） | 中国/米国 → 顧客 | 7-14日 | 海外ECでは標準的 |
| 日本 | 中国 → 日本 | 10-20日 | 「海外直送の本格フレグランス」 |

### 損益分岐

```
月額運用コスト:       約¥15,000（Shopify Basic + LLM API）
香り製品1個の利益:    約¥2,000-3,000（マージン40-60%）
損益分岐:            月5-8個
```

## 商品キュレーション戦略

ノーブランドの安い商品を大量に並べるのではなく、品質が担保されたブランド品を厳選する。

### 選定基準

| 基準 | 理由 |
|------|------|
| 知名度のあるブランド | 品質が市場で検証済み |
| レビュー評価が高い | 返品リスクが低い |
| 成分・原材料が明記 | 記事コンテンツと連動（推薦に根拠） |
| 香りファミリーの網羅性 | 診断結果に対応する商品が必要 |

### 差別化

```
一般的なドロップシッピング:
  何千点を並べる → 顧客が選ぶ → 品質バラバラ

kaoriq:
  厳選した数十点 → 性格診断で3本に絞る → 「なぜこれが合うか」を説明
```

数ではなく、推薦の精度と説明力で勝負する。

## 商品カテゴリの段階展開

| Phase | 商品 | 法的分類 | 規制 |
|-------|------|---------|------|
| 1（現在） | キャンドル、ディフューザー、お香、ルームフレグランス | 雑貨 | なし |
| 2（将来） | 香水（肌に付ける） | 化粧品 | 薬機法 |
| 3（将来） | OEMオリジナルブランド | 化粧品 | 薬機法 + 製造販売業許可 |

## 法務チェックリスト

### 必要なもの

- [x] 開業届（既出）
- [ ] 特定商取引法に基づく表記（サイトにページ追加）
- [ ] プライバシーポリシー（サイトにページ追加）
- [ ] 返品ポリシー策定

### 不要なもの（Phase 1）

- 化粧品製造販売業許可（空間用雑貨は対象外）
- 古物商許可（新品のみ）
- PSCマーク（対象外）
- 輸入ライセンス（通常通関のみ）

### 推奨

- PL保険（年2-5万円、キャンドル・お香の火気リスク対策）

### マーケティング上の注意

- 「お部屋用」「空間用」→ 雑貨（許可不要）
- 「肌に付ける」「ボディに」→ 化粧品（許可必要）
- 同じ商品でもマーケティング文言で法的分類が変わる

## 自律運用アーキテクチャ

### 全自動フロー

```
harness-ops (Claude)
├── 商品選定
│   └── 評価・レビュー・成分データで自動スクリーニング
├── 商品登録
│   └── Shopify Admin APIで登録・説明文生成
├── コンテンツ生成
│   ├── 記事生成（1日1-2本）
│   └── 4PF同時投稿（自ドメイン / Medium / Pinterest / Tumblr）
├── パーソナライズ推薦
│   ├── persona-manager → 性格診断
│   └── 59,000本DB → 「あなたに似合う理由」生成
├── 注文処理
│   └── DSers自動注文確定
├── カスタマー対応
│   └── AIチャットボットで一次対応
└── PDCA
    ├── Observer: GA4 / Pinterest / Shopify数値観測
    ├── Strategist: カテゴリ・記事の戦略判断
    └── Marketer: 施策実行
```

### kenさんの関与

定期的な商品選定と月次の方針判断のみ。日常運用はすべて自動化。

## マルチプラットフォーム戦略

### 投稿先

| プラットフォーム | 役割 | 言語 |
|----------------|------|------|
| kaoriq.com | canonical（本丸）、LLMO最適化 | JA/EN |
| Medium | 英語圏リーチ + DA借用 | EN |
| Pinterest | ビジュアル検索 + 香り最強チャネル | JA/EN |
| Tumblr | アート・ライフスタイル層 + 被リンク | EN |

### 投稿フロー

```
harness-opsが1日1記事を生成
  ├── kaoriq.com（canonical_url、LLMO最適化）
  ├── Medium（記事全文、canonical_url付き）
  ├── Pinterest（記事から画像ピン生成 + kaoriqリンク）
  └── Tumblr（記事全文、canonical_url付き）

1日1記事 × 4PF = 月120投稿
全投稿がkaoriq.comへの被リンクになる
```

## 出口戦略

| 出口 | 条件 | リターン |
|------|------|---------|
| ドロップシッピング利益 | 月5個〜で黒字 | 月数万〜数十万の継続収益 |
| OEM自社ブランド | トラフィック安定後 | 利益率さらに向上 |
| サイトM&A | 月間PV 100万規模 | 数百万〜数千万円の一括売却 |

どの出口に転んでも回収できる構造。失敗しても金銭的損失はShopify月額のみ。

## 循環エコシステム

```
① 本（知識の権威性）
   ↓ 「この人は香りをデータで理解している」
② 記事（LLMO + Pinterest）
   ↓  集客
③ Shopify（香り製品販売）
   ↓  実績・データ
④ 実践事例として本に還元
   ↓
① に戻る（改訂・新刊）
```
